# ADR-0004: Unfold Beacon notification service (test scaffold)

## Context

This record exists **only for automated and manual testing** of ADR ingestion, graph generation, and multi-service diagrams. It introduces a **fictional** new runtime component so tooling can exercise edges, labels, and service counts without shipping code.

## Decision

**Fictional change (not implemented):** add **Unfold Beacon**, a small internal service that subscribes to the same GitHub webhook feed as Unfold Engine and sends human-readable summaries when ADRs change (e.g. to Slack or email). Beacon does **not** write `architecture.json`; it only consumes repository metadata and optional Engine build artifacts for richer messages.

**Unfold Beacon (fictional)**
- Internal service
- Stateless worker (e.g. Node or Go) on a managed queue consumer
- Triggered by GitHub `push` events filtered to `adrs/**`
- Calls no user-facing API; optional read-only fetch of `architecture.json` from the default branch for diff context

Relationships to existing fictional components: **GitHub** → webhook → **Unfold Engine** (writes graph) and **Unfold Beacon** (notifications only); **Unfold Lens** unchanged (reads `architecture.json`).

## Consequences

- **Test-only:** no code, infra, or secrets are implied by this document
- If this were real: another deployable to operate and monitor; clear separation keeps Lens decoupled from notifications
- Safe to delete or replace when tests no longer need a sample “new service” ADR
